Field Manual

The complete reference for RUMPUS ENGINE — every tab, section, fold and slider in the level editor, plus the systems behind racing, vehicles, signals, multiplayer and sharing. For guided walkthroughs with one-click example projects, open Help & tutorials on the game's home menu instead.

BasicsGetting started

RUMPUS ENGINE is a single-file browser game studio — build worlds, play them (FPS, racing, top-down, side-scroll), and share them. Nothing installs; the whole game is one page.

The home menu

  • Deploy — play the current level (your autosaved work, a loaded file, or the default arena). Its equal twin Build / edit level opens the editor on the same level.
  • Multiplayer — host or join a room (see Multiplayer). Community levels — browse and play what other creators have published.
  • Play campaign / Load campaign — play a sequence of chained levels with carry-over loadout (in the footer strip).
  • Instructions — a quick controls card. Help & tutorials — guided tutorials with loadable example projects. Field manual — this page. Credits — engine libraries plus the asset attributions collected for the current level.
The build number is shown at the bottom-left of the home menu. If a fix doesn't seem to be live, hard-refresh (Ctrl+Shift+R) and check that number first.

Gamepads and touch are supported: plug in a controller (press ] to switch between detected pads), or on a phone use the on-screen joysticks and drive cluster — pause → ✿ Customize controls to move or resize any on-screen button.

Rebinding: pause → ⌨ Keyboard bindings to remap movement and every action (click an action, press the new key; picking a key already in use swaps the two; Reset restores defaults). Weapon slots 1–5, Esc and the editor's own keys stay fixed, and on-screen prompts keep showing the default keys.

Controller menus: every menu is playable from the pad — D-pad or left stick moves the focus ring, A selects, B backs out, and left/right tunes sliders and dropdowns. The first press lands on Deploy.

Controller: Start pauses (solo) or opens the match menu (multiplayer). D-pad: up = inventory, down = map, left = melee, right = flashlight. In the supply cache, D-pad picks, A buys, B/Y closes; on the upgrade screen, bumpers or D-pad focus a card and A takes it (keyboard: press 1–3). Look sensitivity, aim sensitivity, stick deadzone and invert-Y live in the pause menu once a pad is connected.

Touch: besides the joysticks — CROUCH toggles, MELEE swings, BUILD opens the deploy radial (tap a slice to place), and hold USE to grab or drop a physics prop. LIGHT, BAG and CHAT appear automatically when a level has a flashlight, an inventory, or you're in a multiplayer room.

Editor referenceBuild tab

Add shape

+ Box, + Sphere, + Cylinder, + Cone, + Ramp, + Stairs, + Dome, + Tube, + Ring drop a primitive in front of the camera. Every shape gets the full Material editor. The first four use exact box/sphere colliders; the construction shapes (Ramp/Stairs/Dome/Tube/Ring) collide with their real triangles, so you can walk and drive up a ramp or staircase, shelter under a dome (it's hollow — visible from inside), run through a tube, or hop through a ring. Like every primitive they're authored to a 1×1×1 footprint with the base at floor level, so scaling and floor-snapping behave exactly like the box: stretch a Ramp long and low for a drivable slope, or scale Stairs up for building-sized steps. A Pillar and the track pieces are also primitives under the hood.

Track builder

Snap-together race track pieces — covered in depth in Track builder & racing. With pieces placed, the Track appearance panel restyles the whole course at once: road colour, roughness and an optional road texture (with a tiles dial), painted-line colour, the kerb stripe pair, and the barrier colour. Saved with the level; Reset to stock returns the classic asphalt look.

Search free models

One search box finds free 3D models on Poly Pizza (no key needed) and Sketchfab (needs a free API token — enter it once, everywhere else shows “Token saved · change”). Click a result to place it. Every searched asset is recorded for the level's Credits screen automatically.

Object & selection

With something selected, this section shows: swap/replace model, and the eight folds that give a prop behaviour — animation, locks, signals, dialogue, vehicle, joints, mechanisms, physics & destruction.

Material

For shape primitives: colour, texture map with U/V tiling and rotation, shininess, metalness, emissive glow — and Opacity. Opacity below 1 makes the shape see-through: glass = low opacity + high shininess, frosted plastic = mid opacity + low shininess, and a faint pane at the minimum (0.15) reads as a force field. A transparent prop stays completely solid — you can stand on a glass floor and bullets still hit it. Opacity saves with the level and carries through share links and multiplayer like every other material setting. Imported models keep their own materials — the section says so instead of hiding.

Transform & Handles

Transform is the numeric version of the gizmo: exact position, rotation and scale fields. Handles pick the gizmo mode (move / rotate / scale — also 1/2/3) and proportional scaling (4).

Groups

Every prop's Tag now sits at the top of Object & selection (and stays in the Signals fold — they edit the same tag). Ctrl+Shift+C collapses every editor section at once.

Outliner (scene hierarchy)

Press O (or the list button on the editor toolbar) for the Outliner — the whole placed scene as one panel: props, lights, enemy spawns, turrets and pickups, each with its tag. As levels grow this is the fastest way around:

  • Search matches names, tags and folder names; the tag dropdown narrows the list to one tag.
  • Folders: select some props, then use the Move → dropdown in the Props group to file them into a folder (or create a new one) — or just drag a row onto a folder (drop on the Props header to un-file it). Folders save with the level. Double-click a folder to rename it; a folder's eye/padlock hides or locks everything inside at once.
  • Eye hides an object while editing only — deploys and play always show everything. Great for getting a roof out of the way.
  • Padlock locks an object: viewport clicks and marquee boxes pass straight through it, so a big backdrop stops stealing every click. A locked object can't be selected or moved at all until you click its padlock again.
  • Select all of a type: the grid button on a type header (or a folder) selects everything it contains — filter first to select just that slice (e.g. every prop tagged door).
  • Double-click a prop or light to name it; names save with the level and make search actually useful.

Select multiple props and press Ctrl+G to group them — the group moves, rotates, scales and duplicates as one. Ctrl+Shift+G ungroups.

Prefabs

Turn any configured selection into a reusable component — a door with its trigger, an elevator, a trap, a whole furnished room. Select the props, type a name in the Prefabs section and hit Save selection: the capture keeps everything — signals, mechanism animation, physics and joints, vehicle tuning, locks, dialogue, materials. Place drops a copy 8 m ahead of the camera; every copy arrives grouped (moves as one) and heights are terrain-relative, so a room built on flat ground sits correctly on a hillside.

Your prefab library lives in your browser and follows you across levels — and every level embeds the prefabs it uses, so a shared or community level brings its components along (they join the library of whoever opens it, never overwriting same-named prefabs of their own).

Editing a prefab: change any placed copy — move parts, retag, recolor, even add new props to its group — select it and press Update on that prefab's row. The definition rewrites and every other copy in the level follows. Parts you deliberately changed on one specific copy (a raised panel, a renamed tag) are detected as local overrides and left exactly as you authored them — the toast reports how many were kept. Moving a whole copy around the map never counts as an override. Detach disconnects a copy so future updates skip it; × removes a prefab from your library (placed copies stay in their levels).

Editor referenceObject folds — giving props behaviour

Select any prop on the Build tab and open Object & selection. Eight folds, in panel order. (Signals and Mechanism get their full treatment in Signals, locks & mechanisms; Vehicle in Vehicle tuning.)

Animation

Plays a model's built-in clips. Pick the clip (or all clips at once), then when it plays — Auto (from level load), E Activate (player interacts), or Signals only (waits to be targeted) — and how: ⟳ Loop, ⇄ Ping-pong, or → Once. Built-in shapes have no clips (use Mechanism to move those); for models without animations, the Sketchfab search has an "Animated only" filter.

Lock

Lock picks the required key (red / blue / gold / green); pressing E on the prop then needs that key in hand — place a matching key pickup pad somewhere findable. Consume key on unlock spends the key (for one-key-one-door puzzles; off = a master key). A prop unlocks once per deploy, for everyone in MP.

Dialogue (NPC)

Name (shown over the prompt) + one line of dialogue per row (up to 12 × 200 chars); the player presses E to advance. The first line also fires the prop's On E signals — so a conversation can set the objective, drop a checkpoint, or open a door as it starts.

Joint (physics)

ControlWhat it does / how to use it
Physics joint (hinge)Hinges the prop so it swings under physics. The prop must also be Dynamic (Physics fold) or the joint does nothing — the editor warns.
Anchor toBlank = pinned to a fixed point in space (doors, see-saws). A prop tag = chained to that body — pendulums, rope bridges, and towed trailers (anchor to a drivable car's tag, keep axis Y, put the hinge at the trailer's front tongue; the car pulls it with real physics).
Hinge axis + X/Y/ZY = door, X = see-saw, Z = paddle. The offsets move the hinge point from the prop's origin — e.g. half the width so a door hinges at its edge. The cyan marker shows the live hinge point in the scene; the ring + line show the swing axis.
Limit the swing angleMin/Max (±180°) — stop a door at 90°, a see-saw at ±25°.
Trailer tracking0–1: simulates tyre grip so a towed trailer trails straight instead of swinging out (0 = free hinge). It still yields in turns — raise it if the trailer whips around, lower if stiff. It centres on the angle it has at Deploy, so place the trailer in line behind the car.
Motor speed / strength±720 °/s drives the hinge (0 = free swing) — powered wheels, auto-closing doors, spinning hazards.

Joints take effect on Deploy.

Physics & destruction

Weight0.05–200Heavier = harder to knock around, throws shorter, hits harder under Crushing-props rules.

Editor referenceWorld tab

Sections: Environment · Auto-generate · Zones

Environment

Movement

Global player physics: Walk speed (4–60) · Run speed (4–100) · Crouch speed (1–40) · Jump (0–30) · Launch power (0.25–4, jump-pad multiplier) · Gravity (5–80) · Eye height (0.4–20).

Lighting — control by control

ControlRangeWhat it does / how to use it
Sun (directional)0–4The main shadow-casting light's strength. 0 kills direct light entirely (flat, moody); ~0.9 default; 2+ is harsh desert noon. This is the only light that casts shadows.
Sun direction °0–360Spins the sun around the map — every shadow swings live as you drag. Match it to the visible sun in your HDRI sky, or aim shadows across your racing line for drama.
Sun height °5–89Angle above the horizon. 10–20° = long golden-hour shadows; 60°+ = tight noon shadows. Floored at 5° because a horizontal sun degenerates the shadow map.
Sky (ambient fill)0–3Hemisphere fill from above — lifts everything the sun doesn't reach. Low values make shadows read near-black.
Brightness lift0–2A flat ambient boost on top of everything. Use sparingly — it flattens contrast fast. Prefer raising Sky fill first.
Sun / Sky light colourcolorTints direct light and the ambient fill separately. The default fill leans cool teal; pick a neutral grey to remove the cast, warm amber for sunset scenes.
Day/night cycleon/offThe sun crosses the sky during play: dawn → noon (70° up) → dusk → midnight (below the horizon). Twilight ramps smoothly, the horizon crossings go golden, nights cool to a blue moonlight floor — dark but never black. Fog and flat skies dim after dark, and cars with headlights switch them on automatically when entered at night. Turning it off (or opening the editor) restores your authored sun instantly.
Cycle length s30–1200One full day+night. 30–60s = timelapse showpiece; 240 (default) = a day per 4 minutes; 1200 = a slow ambient drift across a long session.
Start atDawn/Noon/Dusk/NightWhere in the cycle each deploy begins. Start at Dusk for a level that slides into darkness mid-mission.

Tip: the day/night cycle can't dim an HDRI sky texture (only fog, lights and flat-color skies) — for convincing nights, use a plain sky. Sun direction still sets where dawn breaks when the cycle is on.

Camera & view

Field of view (50–110) · View distance (60–2000 — the hard cutoff where geometry stops drawing; raise it for big racing arenas) · Fog density (lower = see farther into the dark) & colour · ground grid toggle. Depth of field: Focus distance = how far away stays sharp, In-focus range = the depth kept sharp around it, Blur strength = how soft everything else goes. Post-processing: Bloom + Bloom threshold (how bright a surface must be before it glows), Motion blur (trail strength), Vignette, Film grain, Saturation, Contrast — they all compose, DoF included, and they save with the level (the on/off switch is each player's preference).

Sky

Browse Poly Haven HDRIs in-editor or paste any equirectangular .hdr/.jpg URL (an .hdr lights reflections too); presets Park / Mountain / Room / None (None = flat fog-colored sky). Reflection strength (0–3) sets how strongly the sky shows up on metallic floors and walls — 0 for fully matte scenes. Sky rotation° spins the panorama to aim its sun or horizon features.

Weather — control by control

ControlRangeWhat it does / how to use it
None / Rain / Snow—Rain renders as line streaks that lean into the wind and stretch with fall speed; snow as soft round flakes that drift and sway per-flake. Both are one recycled particle layer riding the camera — density halves on phones automatically.
Amount0.1–1Particle density. 0.2 = light drizzle / flurries; 1 = downpour / blizzard.
Size0.5–2Streak length (rain) or flake size (snow), applied live as you drag.
Wind0–30 = dead calm (rain falls straight, snow just sways); 3 = storm — rain lies hard over, snow streams sideways.
Wind dir °0–360Which way the wind blows. Line it up with your waterfall spray, flag props, or just against the player's line of travel for drama.

The layer follows the camera and falls indoors too — keep sheltered scenes on None, or accept it as "atmosphere". Storm recipe: Rain, Amount 1, Size 1.4, Wind 2.5, day/night cycle parked at Dusk.

Floor & terrain

Floor colour / roughness / metalness, texture with tile W/H and rotation, and a texture search. Terrain: Height (0–18) and Detail (1–8) with Generate / Regenerate / Flatten, plus a terrain brush painted directly onto the ground — Raise / Lower / Smooth sculpt the height, Paint blends materials, and Scatter plants props. A colour-coded cursor ring follows your pointer showing the brush radius; in Paint mode the inner ring is where full strength ends and the soft edge begins, in Scatter mode it shows the planting spacing.

Material painting (the Paint brush mode)

Make the base floor your ground material (say, tiled grass), then paint up to three more textures straight onto it — a dirt path, a stone plaza, scorched earth. Pick a Layer, give it a texture (URL or the built-in free-texture search) and a Tiles count, then drag on the floor. It works on flat floors and sculpted terrain alike, and the result saves with the level (share links, gallery, multiplayer).

ControlRangeWhat it does / how to use it
Layer 1 / 2 / 3—Which paint layer the brush writes. A ● marks layers that already have a texture. Where layers overlap, the higher-numbered layer paints over the lower.
Tiles2–120How many times this layer's texture repeats across the arena — match it to your base floor tiling so the scale reads consistently.
Brush size2–40Radius in metres. Big for clearings, small for a winding footpath.
Soft edge0.05–0.95The blend band at the brush rim. Low = crisp painted edge; high = the grass-fades-into-dirt look. This is the "soft edges" dial.
Flow0.05–1How much each pass lays down. Low flow + repeated strokes builds up like an airbrush — great for subtle wear down the middle of a path.
Erase—The same brush removes the selected layer's paint instead of adding it.

Path recipe: base floor = grass texture; Layer 1 = dirt, Tiles ≈ your floor tiling, Brush size 4–6, Soft edge 0.6, Flow 0.4 — then drag your route in two or three slow passes.

Landscape scatter (the Scatter brush mode)

Plant randomized props by dragging: trees, bushes, rocks — anything. Fill up to three slots, either from the free-model search (credited automatically) or with Use selected prop, which clones whatever's selected at its exact scale — including built-in shapes. Each slot has its own Size dial: for searched models it's the real-world size in metres (default 7 — tree height; crank it to 25 for giants), for cloned props it's a multiplier on their captured scale. Each pass drops Density (1–6) props at random spots inside the brush circle, each randomly rotated, varied by Size jitter (±0–60%), and sat on the terrain — a slow drag along a hillside plants a treeline.

Scattered props are normal props: marquee-select to move or delete them, undo works, and they save with the level. Budget note: each searched model adds a draw call — a few dozen trees is fine, hundreds will cost frames. Scattered primitives (rocks from spheres, fence posts from cylinders) are basically free thanks to instancing. Forest recipe: slot 1 = a tree, slot 2 = a second tree variety, slot 3 = a bush; Density 3, Size jitter 0.4, Brush size 15 — then wander the map edge in one slow pass.

Walls & arena

Show boundary walls (off = invisible but still solid), wall material/texture controls, and Arena size (15–2000) — half-width of the playable square. Racing levels usually want 300+.

Audio

A looping music track plus per-event sound rows: grenade explosion, coin pickup, powerup pickup, jump, hit marker, enemy kill (with per-enemy-type overrides), player hurt, and the car set — engine loop, brake, skid, boost. A built-in Freesound browser searches free sounds; each pick is credited automatically.

Auto-generate

Procedurally builds a starting layout (cover, buildings, scatter) you then edit by hand. There's also an AI scene builder — describe a scene in a sentence and it searches and places matching models (works best with a Poly Pizza or Sketchfab key).

Zones

Paintable volumes under World → Zones — pick a type at the top of the section, then + Add (at me) drops one where you stand. Audio zones (positional sound while inside a radius: URL, volume, loop), Death zones (instant kill between Base Y and Base Y + Height — lava, pits, out-of-bounds), Jump pads (launch on contact, scaled by the Movement fold's Launch power), Ladders (climbable volumes: W up, S down, jump to hop off; Facing sets which side you step off at the top), and Fire zones (damage per second inside, with a full particle look: density, size, hue, turbulence, wind, saturation, sparks, smoke mode — hue 30 is natural fire, 210 blue, 285 purple; saturation 0 turns it into grey smoke).

Water zones — control by control

ControlRangeWhat it does / how to use it
Radius1–120Footprint of the water cylinder. Puddle ≈ 2, pond ≈ 8–15, lake 30+. Chain several round zones for a winding river.
Depth0.05–20Water fills from Base Y up to Base Y + Depth; the animated surface sits on top. 0.05–0.3 = a glossy visual puddle; 0.3–1.2 = wading (players slowed, gravity still applies); 1.3+ = swimming (buoyancy, Space strokes up, C dives, the screen tints while your eyes are under). Deep water also brakes falls — jumping off a cliff into a deep pool is safe by design.
Base Y−20–40The floor of the water volume. For a lake in a sculpted basin, set it at the basin floor so the surface lands at natural shore height.
X / Z±2000Exact placement (the zone also drops where you stand when added).
Water colorcolorThe deep tint. Murky green for swamps, deep teal for lakes, near-black for horror. The surface also picks up a sky tone via its fresnel and a glint from your actual sun direction — and the whole body follows the scene lighting: Sun/Sky sliders brighten or darken it, and the day/night cycle dims lakes and falls through dusk into a faint moonlit sheen.
Opacity0.1–0.95Low ≈ clear shallows (you can see the bottom); high ≈ opaque depths. Pair low opacity + tiny depth for puddles.
Waves0–2.5Ripple strength. 0 = glass (mirror-calm), 1 = default chop, 2+ = agitated. Purely visual.
Flow dir ° / Flow push0–360 / 0–12Turns the zone into a stream: the current pushes swimmers along Flow dir at Flow push strength, and the ripples visibly scroll with it. ~3 = lazy river you can swim against; 8+ = rapids that sweep you away. 0 push = still water.

Water is mechanical, not just visual: cars slow in it (light splash → 45% of top speed in the shallows → 18% in deep water) and throw bow spray; dynamic props bob to the surface and float; enemies and bots wade at roughly half pace — funneling hostiles through a river is a legitimate defensive design. One draw call per surface, so lakes are phone-friendly.

Waterfalls — control by control

They live inside the Water tool, below the zones list. Each fall is two layered sheets of streaming columns (the back layer runs slower and dimmer for depth) plus a churning foam pool.

ControlRangeWhat it does / how to use it
X / Z / Top Y—Where the lip of the fall sits. Put Top Y at the cliff edge height.
Drop1–60How far it falls. Match it to the cliff so the foam pool lands at the base (pool sits at Top Y − Drop).
Width0.5–60Sheet width. 1–2 = a rivulet, 6 = default cascade, 20+ = a broad curtain.
Facing °±180Rotates the sheet — set it flush against the cliff face it pours over.
Tint / Opacity—Water color and sheet density. Low opacity + slow speed reads as a gentle veil; high opacity + fast speed as heavy whitewater.
Speed0.1–4How fast the water streams down (also scales the streak animation).
Splash0–2Foam pool intensity — how hard the base churns.
Pool width0.2–2.5Foam pool size as a multiple of the sheet width (default 1.15 spans the full fall). The pool is elliptical — wider along the sheet than out from it.

Standing in the sheet pushes you down into the plunge pool — pour a fall into a deep water zone and the combination is safe to ride down; onto dry ground, the landing is on you.

Editor referencePlayer tab

Sections: Handles · Object & selection · Transform · Characters

  • Player avatar — import any model as the third-person body; scroll in the preview to zoom.
  • Player start — drag the blue marker to set where the run begins; Facing° (−180…180) sets the initial look direction.
  • Characters — the roster of player & enemy character models, with animation-slot mapping (clips are grouped by taxonomy: locomotion, combat, hits, deaths…). Characters picked here appear in multiplayer's CHARACTER menu too.
  • Slide on touch — tap CROUCH while sprint-running; a standing tap still toggles crouch. On a controller the bag and item inspector are fully navigable: stick/D-pad moves the ring, A opens/uses (including Use — hold & place), B backs out.
  • First-run tutorial — your first solo deploy shows a small coach pill (Move, Look, Fire, Jump, Sprint, Build menu); each step ticks off when you do it, the wording matches your input device, and the ✕ dismisses it for good.
  • Model loading bar — picking a Sketchfab or Poly Pizza model shows a live progress pill (percent and MB) while it downloads, so you know it's coming.
  • Joint tweaks — models that import with bent wrists, A-pose arms or twisted forearms can be fixed in-app: pick a bone on the Player tab (Joint tweaks), nudge X/Y/Z in degrees, and the offset rides on top of every animation. Live in the preview, saved with the character.
  • Auto-rigger — a static, unrigged humanoid model (T-pose or A-pose) becomes a fully animated character without leaving the game. Load the model on the Player tab and press Auto-rig model (T-pose): a Mixamo-style modal shows the model front-on — click chin → wrists → elbows → knees → groin (with Use Symmetry on, one click per pair fills both sides; drag to turn, scroll to zoom, click a step row to re-place it). AUTO-RIG fits a full humanoid skeleton to your markers, computes skin weights automatically (weights blend smoothly across knees and elbows and are diffused along the mesh surface, so deep bends curve instead of creasing), and switches the animation library on — the model walks, runs, crouches and shoots immediately. The markers are a few numbers saved with the level, so the rig rebuilds on every load; works at any model scale. Blocky/stylized models deform best — fine-tune joints afterwards with Joint tweaks if needed.
  • Animation editor — author your own skeletal clips entirely in the browser (desktop only; no Blender or Mixamo needed). On the Player tab press 🎬 Animation editor: the model opens in a full-screen workspace with a bone tree (friendly names, search), an orbit viewport (click a joint to select it, drag the selected joint to rotate — hold X/Y/Z for a local axis, Shift-drag the hips to move them), an inspector (exact angles, reset, key buttons) and a timeline (scrub, play with Space, drag keys, snap-to-frame, K keys the selected bone, Shift+K keys the whole pose, Auto-key writes keys as you drag). Clips save into the level (share codes, multiplayer and campaigns carry them) and appear in every per-state clip picker next to built-in and library clips — or assign one directly from the editor's Assign panel (idle, walk, reload, death, any state). Because clips are stored against the engine's normalized skeleton, the same clip retargets onto any humanoid model. Ctrl+Z undoes inside the editor; deleting a clip warns where it's assigned. Production tools: Mirror pose / Mirror clip (left↔right), Copy/Paste pose and Copy/Paste keys at the playhead, a per-clip Curve mode (smooth / linear / step), Ghosts (onion-skin skeletons at the previous and next keys), Export/Import of .rumpusanim files to share clips between levels and creators, and timeline events — fire a sound (footsteps, casings, reload clicks) or a → Logic event (melee hit-windows, scripted beats) at any moment of the clip; they show as orange ticks on the timeline and fire wherever the clip plays.   • Built-in animation library — pick Standard (43 clips) or Pro (120 clips — every animation slot filled: 8-direction locomotion, climbs, turns, dodges, five hit reactions, two deaths) under Animation states and RUMPUS ENGINE retargets the CC0 clip pack (by Quaternius) onto your model's own skeleton. SystemsVehicle tuning — the deep dive

Any prop can be a car: select it → Object & selection → Vehicle (drivable) → tick Drivable vehicle (this also makes the prop non-dynamic — the car system owns its physics). A units toggle switches every speed field and the in-game speedometer between km/h and mph.

How the handling actually works — three separate systems, not one "handling" number: Braking fires when your throttle opposes your direction of travel: you slow with real stopping distance (an ABS-like ease-off in the last couple of m/s) and only then does reverse engage — you can never snap from forward into reverse. Cornering grip (G) is the lateral force the tyres hold: your achievable turn rate shrinks as speed rises, so corners widen like a real car (understeer) — but steering never drops below 35% of what you ask, so it never "bricks". Flooring the throttle mid-corner spends tyre on acceleration instead of cornering (a traction circle). Grip is how fast your direction of travel chases your heading: low grip = the car slides and drifts; the handbrake cuts it to ~16% for deliberate slides; pushing past ~90% of the cornering limit steps the rear out progressively, and counter-steering recovers grip 1.4× faster.

SystemsSignals, locks & mechanisms

These three folds are RUMPUS ENGINE's wiring: Mechanism makes a prop move, Signals connect cause to effect, and Lock gates it all behind keys. Together they build doors, elevators, traps, switch puzzles and win conditions.

Mechanism (the “xa” system)

Enable Animated mechanism on a prop, then set what it does — Move X/Y/Z (m), Rotate X/Y/Z (°), Scale X/Y/Z, a Pivot offset, and Duration (s):

  • Playback: ⟳ Loop · ⇄ Ping-pong · → Once.
  • Trigger: Auto (runs on level start) · E Activate (player presses E) · Signals only (waits for a signal).
  • Waypoint path — add waypoints and the prop travels through them (overrides the Move offset): patrol platforms, cable cars, roaming hazards.

Signals

Making a prop usable with E is one checkbox now: Interactable (E) sits beside the Tag at the top of Object & selection. Tick it and the prop prompts Activate and fires its On E signals — no mechanism required. And a signal's target can simply be this (the first entry in the target drop-down): the prop the signal lives on, no tag needed. So "press E on the door, the door opens" is: Interactable ✓ + one signal On E → Toggle → this.

The advanced gates

  • Needs N different senders — the receiving prop only reacts once N different props have signalled it (the same sender firing twice counts once). Both pressure plates; all three generators.
  • Needs item — a rule can require the player to be carrying an inventory item, with an optional consume: “On E, if carrying the fuse, do Open — and spend the fuse.” A locked prop without the item shows a LOCKED toast naming what's missing.
  • On object placed extras — filter by a from tag (only the tagged object triggers it, blank = any physical prop), choose Touching it vs Inside it, and optionally consume the placed object (it vanishes when it lands — sacrificial offerings).

Locks & keys

The Lock fold requires a key (red / blue / gold / green) before E works on the prop, with optional Consume key on unlock. Keys are granted by key pickup pads (one-shot — they don't respawn), appear as HUD chips, and can carry a custom display name set on the pickup.

SystemsVisual logic — the Logic graph

The signal system wires one cause to one effect. The Logic graph (Gameplay tab ▸ Logic graph ▸ Open logic graph) is the layer above it: a full-screen node canvas where you build real game logic — counters, timers, branches, variables, randomness, loops and reusable functions — without writing code. Drag the board to pan, wheel to zoom, drag nodes by their headers, drag an output port onto an input port to wire them, click a wire to remove it. Tag, event-name, animation-clip and sound fields are drop-downs: they list every tag your props and lights already carry (with a count), every event name declared anywhere, every clip name your placed models ship, and every sound URL the level already uses — pick from the list or type a brand-new value.

CategoryNodes
Events (cyan)On start · On event (named — fired by prop signals via → Logic event or by Send event) · Every X sec (optional fire limit) · On enemy killed · On wave start

Reusable functions are the On event / Send event pair: build a chain once behind a named event and call it from anywhere — the calling chain continues after the call. A classic graph: On start → Set score = 0; a pressure plate fires → Logic event "scored" → Change score +1 → Branch score ≥ 5 → Show message → Win level.

The graph runs on the host (its world actions broadcast through the signal system, so multiplayer stays in sync) and pauses while the editor is open — playtest to see it run. Variables and timers reset at every match start. A mis-wired infinite loop can't freeze the game: the graph stops for that frame and a toast tells you to add a Delay.

Rules that shape the run

  • Objective text — the goal banner (players re-read it with J; signals can rewrite it mid-run).
  • Respawn at checkpoint (solo) — death returns you to the last checkpoint instead of ending the run.
  • Wave mode — Random, or Pre-built from your spawn markers (optionally single-pass, no respawns); between-wave upgrade cards in solo.
  • Fall damage — on/off for player and AI, with safe speed and damage scaling. Ragdoll on death. Crushing props — heavy movers deal damage, with min speed.
  • Camera view — how players see your level: First person (the classic), Third-person, Top-down, or Side-scroller. Saved with the level, so it carries through share links, the community gallery and multiplayer.
    • Third-person — an over-the-shoulder chase camera that smoothly follows the player with damping (it eases after sprints, jumps and turns instead of being bolted rigid; aiming tightens it back to exact). Controls and aim stay first-person-style. Tune the framing and the Follow smoothing slider under Player tab → Third-person framing — smoothing is also a player preference, so it applies when anyone toggles third-person from the pause menu too.
    • Top-down — a fixed isometric-tilt camera follows the player; WASD moves on the map (screen-relative), and the mouse steers a cursor: your character faces it, shots hit whatever is under it, and grenades lob onto it (up to the grenade's throw-force range), twin-stick style. Cars keep their full handling — instant chase/racing levels. Camera distance sets the zoom (8–80).
    • Side-scroller — the camera locks to one side and players run along a single lane (pick east–west or north–south). A/D run, Space jumps, and the mouse aims anywhere within the plane (grenades arc to the cursor too). Players are held on the lane they deploy on, so build your level along it — ramps and stairs shine here.
    • Cutscenes, turret seats and the PvP killcam keep their own cameras; the editor always flies free. In both fixed views your character model is visible and enemies aim at it as normal.
  • Hands / unarmed — start with fists only, optionally allow picking up weapons found in the level; the flashlight (brightness, cone, softness, range, colour).
  • Hit numbers — floating damage numbers with font size and hit/kill colours.
  • Multiplayer spawn area — confine MP spawns to a circle or drawn polygon.
  • End-screen messages — custom victory and defeat text.

SystemsCutscenes — shot by shot

Gameplay tab → Cutscenes. A cutscene is a list of camera shots played back-to-back with hard cuts; each shot flies through its own waypoints. Cutscene #1 is the intro (toggle Play intro flythrough on deploy); every additional cutscene is named and fired by a signal's Play cutscene action. Any key or tap skips playback.

Waypoints: fly the editor camera to a spot, click + Add camera waypoint (current view), repeat — 2+ points move the camera; a single point is a held shot. Per point: Go to (jump the editor view there), Set (overwrite with the current view), 👁 (aim the editor at your subject, then click to store a look-target), ⊘ (clear it), ✕ (delete).

SystemsMultiplayer

Peer-to-peer over WebRTC — no accounts, no servers to rent. One player hosts and shares the 5-character room code (or joiners pick from the Open games list); everyone else joins, and the host's level transfers automatically — including custom models and sounds. While a host waits in the pre-game lobby, the room is listed in everyone's Open games list automatically (a tiny lobby directory on rumpusengine.com) and drops off the moment the match starts or the host leaves.

PvP hosts pick bot count (0–7) and difficulty when hosting, or Fill with bots from the lobby. Bots fight like players now: they carry visible weapons, throw grenades (including flushing you out of cover), duck behind cover to reload, and wear random characters from the level's roster so no two look alike.

Match rules — when hosting, the host can restrict which weapons are allowed (melee always stays in) and set starting grenades (0–6) for everyone; the rules reach every joiner and bot automatically.

The lobby is a proper pre-game screen: a live 3D character select (models rotate, colours preview, CHARACTERS and COLORS read as separate groups), a MATCH INFO panel, rotating tips — and creators can attach a lobby backdrop image to a level (Save tab ▸ Title screen ▸ Multiplayer lobby backdrop) that every player sees, softly blurred, behind the lobby. Joiners watching the level transfer see a live LOADING WORLD % instead of a frozen screen. Matches end with a podium ceremony — standings, medals, and an MVP callout in team modes.

What syncs: player movement and state, enemies, kill credit, coins and pickups, prop animations and mechanisms, unlocks and keys, objective progress, race standings, chat, and the host's custom HUD/sounds.

SystemsSaving & sharing

  • Browser save — one save slot plus autosave while editing.
  • .breach export/import — the level (or a whole campaign) as a file. Everything serializes: props, track, vehicle tuning, rules, cutscenes, HUD, and the asset-credit list. The file is plain JSON inside, and old .json exports import forever.
  • Share link — the entire level compressed into a URL. Anyone who opens it plays it instantly. Models re-download from Poly Pizza / Sketchfab on the recipient's machine (Sketchfab models need the recipient's own free token).
  • Challenge links — from the end screen, share the level plus your score to beat.
  • Campaigns — chain levels with carry-over loadout and inventory; share as one file.
  • Credits — every searched model, texture, HDRI and sound is attributed automatically and travels with the level; the Credits screen shows the per-level list.

The community library

Community levels on the home menu is a gallery of player-built levels hosted alongside the game — browse, then Play one or Open in editor to see how it's built (and remix it; loading is undoable).

To publish your own: editor → Save tab → Submit to community library. That grabs a screenshot of your current view as the gallery thumbnail, asks for a level name and your author name, and sends the level straight in for review — no account of any kind needed. Once it's approved it appears in everyone's gallery, credited to you. (If the submission server can't be reached, the game falls back to the old GitHub submission form, which needs a free GitHub account.) Levels that use Sketchfab models get a needs Sketchfab token badge, since those download with each player's own free token.

ExtrasTips & troubleshooting

  • Sketchfab search says it needs a token — it's a free API key: sketchfab.com → Settings → Password & API. Paste it once in any search box; every other box switches to “Token saved · change”.
  • A shared level is missing models — Sketchfab downloads require each player's own token; Poly Pizza models need none. Prefer Poly Pizza for levels you plan to share widely.
  • The game looks out of date — hard-refresh (Ctrl+Shift+R) and check the build number bottom-left of the home menu.
  • Performance dips — post-FX sheds passes automatically under load, but the biggest wins are yours: fewer point lights, smaller textures, fewer high-poly Sketchfab models. Adaptive resolution (pause menu) trades sharpness for frame rate on sustained drops — turn it off if you'd rather keep full resolution and accept the dips.
  • Edges look jagged / aliased — check Adaptive resolution in the pause menu: when the frame rate struggles it lowers the render resolution, which reads as shimmer on edges; untick it for full sharpness. (Since build 872, Visual effects keeps proper antialiasing — older builds lost it whenever effects were on.)
  • My race rivals won't spawn — rivals need a closed track loop (use Close loop) and a placed drivable vehicle to clone. The Save tab's issues panel calls this out.
  • A mechanism won't fire — check its Trigger: Signals only waits for a signal; E Activate may be gated by a Lock. The signal's target tag must match exactly.
  • I can't find a setting — sections remember their collapsed state; expand the section headers. Selection folds (Vehicle, Signals…) only appear with a prop selected on the Build tab.
